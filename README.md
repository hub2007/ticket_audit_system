# 基于知识图谱与智能体的运维工单智能校核系统

> 湖北省大学生创新训练计划项目 | 湖北经济学院 信息工程学院 | 软件工程

---

## 项目简介

面向学校网络信息中心硬件设备故障报修工单，构建**知识图谱 + 规则引擎 + NLP语义分析 + 双智能体**的语义校核系统。自动对工单进行七层面评分（0-100分），70分以上合格入库，低于70分自动暂存并给出整改建议，支持反复修改直至达标。

**核心目标**：将人工逐条审核转化为系统自动辅助，提升报修效率。

---

## 技术架构

```
前端（浏览器）             后端（Flask）              数据层
┌──────────────┐       ┌─────────────────┐      ┌────────────┐
│ 工单录入表单  │       │ jieba 中文分词   │      │ JSON 知识图谱│
│ 评分结果展示  │←Fetch→│ 四层词库比对     │      │ 四层词库     │
│ Canvas 图谱  │       │ 七层面规则引擎    │      │ 标注数据集   │
│ 库存/暂存区   │       │ 双智能体推理      │      │ localStorage│
└──────────────┘       └────────┬────────┘      └────────────┘
                                │
                         DeepSeek API
                    (ReAct 云端智能体)
```

## 团队分工

| 角色 | 技术栈 | 负责内容 |
|------|--------|---------|
| 后端开发与智能体 | Python + Flask + DeepSeek API | 规则引擎、本地智能体、ReAct云智能体 |
| 前端UI与可视化 | HTML5 + CSS3 + JavaScript + Canvas | 页面布局、校核结果渲染、知识图谱绘图 |
| NLP与词库建设 | jieba分词 + Flask | 四层词库构建、语义相似度计算 |
| 数据与知识图谱 | JSON + 结构化建模 | 知识图谱数据、工单样本收集标注 |
| 测试与协调 | Git + 全流程测试 | 版本管理、功能测试、边界测试 |

## 快速启动

```bash
# 前端（浏览器直接打开即可）
open frontend/index.html

# 后端（需先安装依赖）
pip install flask jieba
cd backend && python app.py
```
